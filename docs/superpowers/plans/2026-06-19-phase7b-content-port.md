# Phase 7b — Full Docs Content Port Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Port the remaining ~279 documentation pages from `apps/docs` (Astro/Starlight) into `apps/website`'s `/docs` MDX pipeline (built in 7a) — glass-styled, fully navigable, with merged install blocks, all images/videos/props, and zero render failures.

**Architecture:** The bulk port is a **deterministic Node transform script** (`scripts/port-content.mjs`) — NOT an LLM fan-out — because copy + regex transforms are mechanical and must be reproducible. LLM agents are reserved for the *verify* phase (render every page, fix content-specific breakage), where judgment is needed. Nav is ported from the Starlight `sidebar.config.mjs`. `MdxGuide` is upgraded to a splat route that also merges a registry install block for component pages.

**Tech Stack:** Node ESM script, `@mdx-js/rollup` (already wired in 7a), react-router-dom 7.18, the 7a shims (`Tabs`/`TabItem`/`Callout`/`PropsTable`/code-block), Vite `public/` static assets.

## Global Constraints

- **Do NOT clobber the 14 hand-fixed backbone guides** already at `apps/website/src/docs/content/*.mdx`: `admin, create, data-providers, data-table, edit, install, list, quick-start-guide, resource, security, show, simple-form, theming, translation`. The port SKIPS these slugs on copy. (Attr-normalization MAY run over them — it is idempotent.)
- **Drop the 7 React-Admin-Enterprise (`enterpriseEntry`) pages** — do not port, remove from nav: `reference-many-to-many-field-base, auto-persist-in-store-base, reference-many-input-base, reference-many-to-many-input-base, reference-one-input-base, realtime-features, soft-delete-features`.
- **`raCoreEntry` sidebar items stay external links** (`https://marmelab.com/ra-core/...`, `target=_blank`) — they have no local content file; do not port content for them.
- **Routing model = flat `/docs/:slug`** for every content page (faithful to the old Starlight URLs; all internal links already rewrite to `/docs/<slug>` via 7a's `remarkRelativeLinks`). For a slug that matches a registry item name, the page also renders a **merged install block** at the top (shadcn-forward: install + docs on one page). The `/docs/components` grid catalog is kept as a visual index; its cards link to `/docs/:slug`; `/docs/components/:name` redirects to `/docs/:name`.
- **Assets serve from `apps/website/public/docs/images/`** (URL `/docs/images/...`). All content image/video refs rewrite `./images/` → `/docs/images/`.
- **Supabase pages keep their nested path** `/docs/supabase/<slug>` (the splat route handles it).
- Every gate green before advancing: `pnpm --filter website typecheck`, `pnpm --filter website build`, Biome.

### Task 1: The content transform script

**Files:**
- Create: `apps/website/scripts/port-content.mjs`
- Test: `apps/website/scripts/port-content.test.mjs`

**Interfaces:**
- Produces: `export function transformContent(raw: string, slug: string): string` and `export const SKIP_SLUGS: Set<string>`, `export const DROP_SLUGS: Set<string>`. `main()` runs when invoked directly.

**Source roots (relative to `apps/website/`):**
- Content: `../docs/src/content/docs/*.md`, `*.mdx`, and `../docs/src/content/docs/supabase/*.md`
- Images: `../docs/src/content/docs/images/**` AND `../docs/public/images/**` → `public/docs/images/**` (preserve subpaths; `content/docs/images/x`→`docs/images/x`, `public/images/x`→`docs/images/x`)
- Props: `../docs/src/content/docs/props/*.json` → `src/docs/content/props/`

**`transformContent` rules (apply in order):**

1. **Split into fence-aware segments.** Track triple-backtick fences (handle leading whitespace: a line matching `/^\s*```/` toggles fence state). Only steps 4–6 transform *non-fenced* text; fenced code is emitted verbatim.
2. **Frontmatter:** keep as-is (it is just `title:` and, in one file, `description:`). Do not touch.
3. **Strip Astro-only imports** (whole line) when they appear outside fences:
   - `import PropsTable from "../../components/props-table.astro";` (4 files)
   - `import { Tabs, TabItem } from '@astrojs/starlight/components';` (quick-start — already a SKIP slug, but keep the rule general)
4. **Asset-import inlining:** for each non-fenced `import IDENT from './images/REST';`, record `IDENT → "/docs/images/REST"`, delete the import line, and replace every non-fenced `{IDENT}` with the string literal `"/docs/images/REST"`. (Handles `<source src={video}/>` in `date-input.mdx`.)
5. **Asset path rewrite** (non-fenced): `./images/` → `/docs/images/` in markdown links `](./images/...)` and `src="./images/..."` / `src='./images/...'`.
6. **Raw-HTML attribute normalization** (non-fenced, only on lines containing an HTML open tag `<[a-z]`): replace JSX-invalid attrs — `class=`→`className=`, `for=`→`htmlFor=`, `autoplay`→`autoPlay`, `playsinline`→`playsInline`, `frameborder`→`frameBorder`, `allowfullscreen`→`allowFullScreen`, `srcset`→`srcSet`, `tabindex`→`tabIndex`, `colspan`→`colSpan`, `rowspan`→`rowSpan`. Use word-boundary matches so `className` is not re-rewritten (idempotent).

**`main()`:**
- For each source content file: derive `slug` (basename without ext; supabase files → `supabase/<base>`). Skip if `slug ∈ SKIP_SLUGS` (14 backbone) or `slug ∈ DROP_SLUGS` (7 RA-EE). Write `transformContent(raw, slug)` → `src/docs/content/<slug>.mdx` (mkdir `supabase/` as needed).
- **Also** re-run steps 4–6 over the existing 14 backbone `.mdx` (read, `transformContent`, write back) to clear their content-debt — idempotent, fixes raw `class=` in `data-providers.mdx` etc.
- Copy assets (both image roots) and props. Use `cpSync(..., { recursive: true })`.
- Log a summary: `ported N, skipped M, dropped K, assets A, props P`.

**Constants:**
```js
export const SKIP_SLUGS = new Set(["admin","create","data-providers","data-table","edit","install","list","quick-start-guide","resource","security","show","simple-form","theming","translation"]);
export const DROP_SLUGS = new Set(["reference-many-to-many-field-base","auto-persist-in-store-base","reference-many-input-base","reference-many-to-many-input-base","reference-one-input-base","realtime-features","soft-delete-features"]);
```

- [ ] **Step 1: Write failing tests** in `port-content.test.mjs` (node:test + node:assert). Cover:
  - fence-awareness: `` ```\n<div class="x">\n``` `` stays `class` inside; a bare `<div class="x">` outside becomes `className`.
  - import stripping: the PropsTable.astro line is removed; `<PropsTable name="X"/>` survives.
  - asset-import inlining: `import v from './images/a.mp4'` + `{v}` → `/docs/images/a.mp4`, import gone.
  - asset path rewrite: `![x](./images/y.png)` → `![x](/docs/images/y.png)`.
  - idempotency: running `transformContent` twice == once.
- [ ] **Step 2: Run, verify fail.** `cd apps/website && node --test scripts/port-content.test.mjs` → FAIL (module not written).
- [ ] **Step 3: Implement** `port-content.mjs` per rules above.
- [ ] **Step 4: Run tests → PASS.**
- [ ] **Step 5: Run `node scripts/port-content.mjs`**, then verify counts: `ls src/docs/content/*.mdx | wc -l` ≈ 14 + ~265 flat, `ls src/docs/content/supabase/*.mdx | wc -l` == 11, `ls src/docs/content/props/*.json | wc -l` == 246, `find public/docs/images -type f | wc -l` == 106. Spot-check `git diff --stat` on the 14 backbone shows only attr/asset edits (no body rewrites).
- [ ] **Step 6: Commit** script + test + generated content + assets + props.
