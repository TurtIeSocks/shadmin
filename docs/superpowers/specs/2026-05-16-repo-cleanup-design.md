# Repo Cleanup — Design

**Date:** 2026-05-16
**Status:** Approved (brainstorming complete, pending implementation plan)
**Scope:** Five-phase repository cleanup covering kebab-case naming, docs audit, story audit, spec replacement, and demo expansion.

## Goals

1. **Consistent kebab-case file naming** across documentation (source already kebab-case in `src/`).
2. **Every public-API component documented** in `docs/src/content/docs/` with usage example, prop table, and per-prop notes.
3. **Every public-API component has a real story** (no `CoverageStory` placeholders) organized under `src/stories/<dir>/` mirroring `src/components/<dir>/`.
4. **Every public-API component has a real spec** (no `expect(true).toBe(true)` placeholders) using story-driven test pattern from `AGENTS.md`.
5. **Every public-API component exercised in `src/demo/`** at least once.

## Definitions

**Public API** — every export from `src/components/<dir>/index.ts` for `<dir>` ∈ {admin, extras, leaflet, supabase, csv-import, mdx-editor, rich-text-input}. Excludes `src/components/ui/` (shadcn primitives). Approximately 250 items.

**Placeholder spec** — file matching exact 12-line stub pattern:
```tsx
import { describe, expect, it } from "vitest";
import { render } from "vitest-browser-react";
import { Basic } from "@/stories/<name>.stories";

describe("<X />", () => {
  it("renders the Basic story", () => {
    render(<Basic />);
    expect(true).toBe(true);
  });
});
```
Currently 119 such files.

**Placeholder story** — file importing `CoverageStory` from `src/stories/_coverage-story.tsx` (renders empty `ComponentGallery` instead of the actual component). Currently 38 such files.

**Sidebar groups (preserved)** — upstream `marmelab/shadcn-admin-kit` ships these 7 groups; no additions or removals to this list:
1. Getting Started
2. Application configuration
3. Page components
4. Data Display
5. Data Edition
6. UI & Layout
7. Misc

**Sidebar groups (per-folder additions)** — one group per specialized folder under `src/components/`:
- Map (Leaflet) → `src/components/leaflet/`
- Extras → `src/components/extras/`
- Supabase → `src/components/supabase/`
- CSV Import → `src/components/csv-import/`
- MDX Editor → `src/components/mdx-editor/`
- Rich Text Input → `src/components/rich-text-input/`

Components in `src/components/admin/` are distributed across the 7 upstream groups by function.

## Shared Infrastructure

Built in Phase 1, consumed by Phases 2–5.

**`docs/scripts/public-api.mjs`** — exports `getPublicApi()` returning the canonical public-API list:
```ts
[{ name: string, slug: string, sourceDir: string, sourceFile: string }, ...]
```
Parses every `src/components/*/index.ts` (excluding `ui/`), reads `export *` lines, derives slug as kebab-case of `name`.

**`docs/scripts/check-coverage.mjs`** — orchestrator that runs all per-phase check scripts:
- `check-sidebar.mjs` (Phase 0, already exists)
- `check-docs.mjs` (Phase 2)
- `check-stories.mjs` (Phase 3)
- `check-specs.mjs` (Phase 4)
- `check-demo-coverage.mjs` (Phase 5)

Wired as `make check-coverage` — single target. Replaces current `make check-doc`. Wired as a `build-doc` precondition so CI fails on drift.

**Custom ESLint rule `no-tautological-expect`** — under `eslint-rules/`. Flags `expect(true).toBe(true)`, `expect(1).toBe(1)`, and similar tautologies. Catches regressions where someone re-introduces the placeholder pattern.

## Phase 2 — Docs audit + typedoc

**One commit:** `docs(audit): fill gaps + add typedoc props + check-docs guard`

### Typedoc integration

`pnpm docs:gen-props` produces `<ComponentName>.props.json` per public component, derived from the TypeScript prop interface via `typedoc --json` filtered to the public exports.

Astro MDX component `<PropsTable name="..." />` reads the JSON and renders the prop table. Doc pages drop hand-maintained tables in favor of types-of-record:
```mdx
import PropsTable from "@/components/PropsTable.astro";

## Props

<PropsTable name="AccessDenied" />
```

Generated JSON is committed to the repo (no runtime gen).

### `docs/scripts/check-docs.mjs`

For each public-API item, asserts:
- Doc file `docs/src/content/docs/<kebab-slug>.md` exists.
- Body has `## Usage` heading.
- Body has `## Props` heading **OR** the component's JSDoc declares zero props (via typedoc JSON).
- Per-prop documented: for each public prop in the typedoc JSON, body has a section matching `` ## `<propName>` ``.
- ≥ 1 fenced code block.
- No `TODO`, `Coming soon`, `Lorem ipsum`, `FIXME` tokens.

Thin-page tolerance: derived per-component from JSDoc length and prop count, not a fixed line minimum. Components with zero public props can omit `## Props`.

### Backfill

For each gap reported:
- Missing page → write new doc following project pattern: `Usage` → `<PropsTable>` → per-prop sections.
- Existing page missing sections → augment.
- Placeholder text → replace.

### Verification

- `make check-coverage` passes (zero gap reports from `check-docs.mjs`).
- `pnpm doc:build` clean.
- Spot-check 5 newly-written pages render correctly in dev server.

## Phase 4 — Spec placeholder replacement

**One commit:** `test: replace 119 placeholder specs with story-driven assertions + no-tautological-expect guard`

### Replacement template

Per AGENTS.md "tests should import and render stories":
```tsx
import { render } from "vitest-browser-react";
import { describe, expect, it } from "vitest";
import { Basic, /* + variant stories */ } from "@/stories/<dir>/<name>.stories";

describe("<ComponentName />", () => {
  it("renders the Basic story", async () => {
    const { getByText, getByRole } = render(<Basic />);
    await expect.element(getByText(/expected label/i)).toBeInTheDocument();
  });

  it("handles <variant prop scenario>", async () => {
    const { getByRole } = render(<Variant />);
    await getByRole("button", { name: /save/i }).click();
    // Behavioral assertion
  });
});
```

### Rules

- Each `it` block has ≥ 1 real assertion (not tautology).
- Reuse story exports — no inline JSX setup duplicating story scaffolding.
- Browser-context tests (`vitest-browser-react`) for any component that hits DOM/styling.
- Pure logic (hooks, utilities) use `jsdom`.

### Coverage criteria per component

- **Atoms** (no behavior): 1 test — "renders Basic story without throwing + asserts on visible output". Direct characterization test, no TDD.
- **Behavioral components** (forms, buttons, lists): Basic render + 1 interaction test per public callback prop (e.g. `onSave`, `onClose`, `onSelect`). TDD per `superpowers:test-driven-development` — write the failing assertion first, then verify it passes against existing implementation.
- **Reference components**: Basic render + 1 test verifying data-fetching wires up (mocked dataProvider call). TDD.

### `docs/scripts/check-specs.mjs`

For each public-API item, asserts:
- `src/components/<dir>/<name>.spec.tsx` exists.
- File does not contain `expect(true).toBe(true)` token.
- File imports from `@/stories/<dir>/<name>.stories`.
- File has ≥ 1 `expect.element()` or `expect(…).to…` call that isn't tautological.

### ESLint rule

`no-tautological-expect` (under `eslint-rules/`) — flags `expect(<literal>).toBe(<same literal>)` patterns. Wired into existing `make lint`. Test for the rule under `eslint-rules/__tests__/`.

### Throughput strategy

Per CLAUDE.md execution-mode memory ("Always use subagent-driven-development"):

Subagent-driven by docs sidebar group — 1 subagent per component-bearing group, total 12 dispatches (Getting Started skipped — it has no components):
- Application configuration
- Page components
- Data Display
- Data Edition
- UI & Layout
- Misc
- Map (Leaflet)
- Extras
- Supabase
- CSV Import
- MDX Editor
- Rich Text Input

Each subagent receives the spec template, the placeholder detection rule, the hybrid TDD rule, and the list of components in its group. Returns: list of files written + per-file test counts. Main thread reviews per-group diff before next dispatch.

## Risk register

| Risk | Mitigation |
|---|---|
| Phase 1 rename breaks external bookmarks | Astro redirects table per renamed slug. |
| Phase 1 internal link rewrite misses an edge case (e.g. links inside code blocks) | `rewrite-doc-links.mjs` skips fenced code blocks; post-run assertion that zero PascalCase refs remain outside code blocks. |
| Phase 2 typedoc gen fails on complex generic types | Fallback: hand-write the prop table for that component; `check-docs.mjs` accepts either a `<PropsTable>` MDX import or a manual `## Props` section. |
| Phase 3 storybook `title:` reorg conflicts with in-flight stories | Phase 3 lands as single PR after Phase 2 merges. |
| Phase 4 subagent dispatches produce flaky browser tests | Subagents use `vitest-browser-react` + `expect.element` async pattern; hybrid TDD catches per-spec issues; main thread reviews per-group diff before next dispatch. |
| Phase 4 12 subagents dispatched serially is slow | Per CLAUDE.md "dispatch parallel agents for independent work" — dispatch 2–3 group subagents in parallel where their target groups don't share state (most don't). |
| Phase 5 demo resources need fake data | Reuse `ra-data-fakerest` + add resource-specific seed in `dataProvider.ts`. |
| `make check-coverage` becomes slow as scripts grow | Each check script returns early on first failure (with full report); main `check-coverage` orchestrator runs them in parallel via `Promise.all`. |
